# Provenance & licensing review (2026-07-12)

Full-tree derivation audit performed prior to public (GitHub) release. Question answered:
is any part of this project based on prior art (copied files/blocks from GitHub, tutorials,
Stack Overflow, vendored libraries)?

## Verdict

**Clean-room, first-party project.** No vendored third-party code, no copied files or code
blocks, no external-source markers, no third-party Python dependencies (stdlib only). All
runtime dependencies (libvirt/virsh, qemu/KVM, squid, nftables, cloud-init, sqlite3,
iproute2; bats/shellcheck as dev tools) are invoked as separate OS programs -- copyleft does
not cross the exec boundary, so none of their licenses constrain this repository. Nothing
constrains the license choice: MIT, Apache-2.0, GPLv3, or proprietary are all available.

## Method

- Marker sweep: `stackoverflow|github.com|adapted from|based on|borrowed|taken from|
  inspired by|originally` -- zero hits in bin/ src/ tests/ etc/ handout/.
- Import audit: Python files import only the stdlib + the project's own `drvps_common`.
- Template inspection (the classic copy-paste surfaces in ops projects) -- each read and
  judged individually, all found original (project-specific content in upstream-mandated
  formats; format-following is functional, not derivation):
  - libvirt network XML (`dr-vps-setup`, `_dr_vps_net_xml`) and domain XML
    (`dr_vps_domain.sh` render): minimal hand-rolled templates; element structure is
    dictated by the libvirt schema; content (ownership marker, `port isolated`,
    serial-log wiring) is project-specific.
  - systemd units (installer heredocs): standard unit format, original semantics/comments.
  - squid SSL-bump + CA generation: standard openssl/security_file_certgen invocations
    (API-dictated); the policy (allowlist-derived nameConstraints, drift-hash CA rotation,
    ownership/coexistence guards) is original design traceable to the CONCEPT docs.
  - cloud-init NoCloud seed: minimal printf-built `#cloud-config`.
  - SQLite schema (`dr_vps_store.sh`): original tables.
- Recipes (`etc/recipes/*.json`): distro cloud-image URL + sha256 pins -- uncopyrightable
  facts; no distro images are redistributed by the repo (operator fetches at runtime).
- Secret sweep before publication: no embedded keys/tokens/passwords (only the project's
  own redaction regex in `dr_vps_snapshot.sh` matches secret-like patterns).
- Test fixtures/goldens: generated by the repo's own scripts (`build-matrix-goldens.sh`).
